Notion de base des réseaux informatiques

RéseauLAN/WANTopologie
▼
1.1 — Définition d'un réseau informatique
Un réseau informatique est un ensemble d'équipements (ordinateurs, imprimantes, serveurs, routeurs…) interconnectés entre eux, capables de communiquer et de partager des ressources (fichiers, imprimantes, connexion Internet, etc.).
💡 Définition clé
Un réseau informatique = plusieurs machines reliées pour échanger des données et partager des ressources.
1.2 — Pourquoi créer un réseau ?
Les principaux avantages d'un réseau :
• Partage de ressources : une seule imprimante pour tous
• Partage de fichiers : accès aux documents depuis n'importe quel poste
• Communication : messagerie interne, appels VoIP
• Sécurité centralisée : gestion des droits d'accès
• Réduction des coûts : moins de matériel nécessaire
1.3 — Types de réseaux par étendue géographique
SigleNom completÉtendueExemple
PANPersonal Area Network~ 10 mBluetooth entre PC et téléphone
LANLocal Area NetworkBâtiment / campusRéseau d'une école
MANMetropolitan Area NetworkVilleRéseau d'une université répartie
WANWide Area NetworkPays / mondialInternet
1.4 — Topologies de réseau
La topologie décrit la manière dont les machines sont reliées entre elles.
TopologieDescriptionAvantageInconvénient
BusToutes les machines sur un seul câblePeu de câblePanne = tout tombe
ÉtoileToutes reliées à un switch centralFacile à gérerSwitch = point critique
AnneauMachines en cercleDébit constantRupture = panne réseau
MailléeChaque machine reliée à plusieurs autresTrès fiableCoût élevé
🌍 Exemple concret

Le réseau de ton lycée utilise une topologie en étoile : tous les ordinateurs de la salle informatique sont branchés à un switch central. Si un PC tombe en panne, les autres continuent à fonctionner normalement.

1.5 — Composants matériels d'un réseau
• Carte réseau (NIC) : interface matérielle d'un ordinateur pour se connecter
• Câble RJ45 (Ethernet) : support de transmission filaire
• Hub : diffuse les données à tous les ports (obsolète)
• Switch (commutateur) : envoie les données uniquement au destinataire ciblé
• Routeur : connecte plusieurs réseaux différents et route les paquets
• Modem : convertit le signal numérique en signal analogique (et inversement)

Modèle OSI et DOD (TCP/IP)

OSIDODCouches réseau
▼
2.1 — Le modèle OSI (7 couches)
Le modèle OSI (Open Systems Interconnection) est un modèle théorique créé par l'ISO pour standardiser les communications réseau. Il divise la communication en 7 couches, chacune ayant un rôle précis.
N°CoucheRôleExemple de protocole
7ApplicationInterface avec l'utilisateurHTTP, FTP, DNS, SMTP
6PrésentationEncodage, chiffrement, compressionSSL/TLS, JPEG, ASCII
5SessionÉtablissement et gestion des sessionsNetBIOS, RPC
4TransportDécoupage des données, fiabilitéTCP, UDP
3RéseauRoutage, adressage IPIP, ICMP, ARP
2Liaison de donnéesAccès au support, adresse MACEthernet, Wi-Fi
1PhysiqueTransmission des bitsCâble RJ45, fibre optique
🧠 Moyen mnémotechnique
Pour retenir les couches OSI de bas en haut : "Physique — Liaison — Réseau — Transport — Session — Présentation — Application"
→ "Pour Les Réseaux Tout S'organise Par Action"
2.2 — Le modèle DOD / TCP-IP (4 couches)
Le modèle DOD (Department Of Defense), aussi appelé modèle TCP/IP, est le modèle pratique utilisé sur Internet. Il regroupe les 7 couches OSI en 4 couches.
Couche DODCorrespondance OSIProtocoles
Application7 + 6 + 5 (OSI)HTTP, FTP, DNS, SMTP, SSH
Transport4 (OSI)TCP, UDP
Internet3 (OSI)IP, ICMP, ARP, RARP
Accès réseau2 + 1 (OSI)Ethernet, Wi-Fi, PPP
🌐 Exemple : Envoyer un e-mail avec Gmail

• Application (Gmail) → utilise SMTP pour composer et envoyer l'e-mail
• Transport (TCP) → découpe l'e-mail en paquets numérotés
• Internet (IP) → chaque paquet est étiqueté avec l'adresse IP destination
• Accès réseau (Ethernet/Wi-Fi) → les paquets sont convertis en signaux électriques ou radio

2.3 — Encapsulation des données
Lors de l'envoi, chaque couche ajoute un en-tête (header) aux données — c'est l'encapsulation. À la réception, chaque couche retire son en-tête — c'est la désencapsulation.
// Encapsulation descendante (émetteur)
Données utilisateur
→ [Segment TCP] : en-tête TCP + données
→ [Paquet IP] : en-tête IP + segment
→ [Trame Eth] : en-tête Ethernet + paquet + FCS
→ [Bits] : 0101110001...

5. Physique : les trames sont converties en bits (0 et 1) envoyés sur le câble ou en Wi-Fi.

💡 Définition
Un protocole de communication est un ensemble de règles et de conventions qui définissent la manière dont les données sont échangées entre deux équipements réseau. Sans protocole, les machines ne pourraient pas se comprendre.

3.3 — Comment fonctionne DNS ?
Le DNS (Domain Name System) est le "bottin téléphonique" d'Internet. Quand vous tapez www.google.com, votre ordinateur ne comprend pas ce nom : il a besoin de l'adresse IP correspondante.
🔍 Étapes de résolution DNS

1. Votre PC demande au serveur DNS local (fourni par votre FAI ou DHCP)
2. Si inconnu, le DNS local demande à un serveur DNS racine (.)
3. Le serveur racine répond : "demande au serveur .com"
4. Le serveur .com répond : "demande au serveur de google.com"
5. Le serveur de google.com répond avec l'IP : 142.250.74.78
6. Votre navigateur utilise cette IP pour se connecter

3.4 — Comment fonctionne DHCP ?
DHCP (Dynamic Host Configuration Protocol) attribue automatiquement une adresse IP à chaque machine qui rejoint le réseau. Processus en 4 étapes : DORA
📋 Processus DORA
Discover → le client cherche un serveur DHCP
Offer → le serveur propose une adresse IP
Request → le client accepte l'offre
Acknowledge → le serveur confirme l'attribution

4.1 — TCP vs UDP
CritèreTCPUDP
ConnexionOrienté connexion (3-way handshake)Sans connexion
Fiabilité✅ Garantit la livraison❌ Pas de garantie
Ordre✅ Données dans l'ordre❌ Ordre non garanti

UtilisationWeb, e-mail, FTPStreaming, VoIP, jeux
🤝 3-Way Handshake TCP
1. Client → Serveur : SYN (je veux me connecter)
2. Serveur → Client : SYN-ACK (ok, je suis prêt)
3. Client → Serveur : ACK (connexion établie)
4.2 — Adressage IPv4
Une adresse IPv4 est composée de 32 bits, écrite en 4 octets séparés par des points. Exemple : 192.168.1.10
Chaque adresse IP a deux parties :
• Partie réseau : identifie le réseau
• Partie hôte : identifie la machine dans le réseau
ClassePlageMasque par défautUsage
A1.0.0.0 – 126.255.255.255255.0.0.0 /8Très grands réseaux
B128.0.0.0 – 191.255.255.255255.255.0.0 /16Moyens réseaux
C192.0.0.0 – 223.255.255.255255.255.255.0 /24Petits réseaux
🔢 Exemple de calcul réseau

IP : 192.168.1.45 — Masque : 255.255.255.0
→ Adresse réseau : 192.168.1.0
→ Adresse de broadcast : 192.168.1.255
→ Plage d'hôtes : 192.168.1.1 à 192.168.1.254 (254 hôtes)

4.3 — Adresses IP privées (RFC 1918)
Ces adresses sont utilisées dans les réseaux locaux et ne sont pas routables sur Internet :
• Classe A privée : 10.0.0.0/8
• Classe B privée : 172.16.0.0/12
• Classe C privée : 192.168.0.0/16

💡 Définition

6.2 — La technologie RAID
💡 RAID
RAID = Redundant Array of Independent Disks : technique qui combine plusieurs disques physiques pour améliorer la performance, la sécurité ou les deux.
NiveauDescriptionAvantageInconvénient
RAID 0Répartition (striping) sur 2 disquesDouble vitesse de lecture/écritureAucune redondance
RAID 1Miroir : copie identique sur 2 disquesSurvie si 1 disque tombe50% de l'espace utilisé
RAID 5Striping + parité (3 disques min.)Bon équilibre perf/sécuritéComplexe à reconstruire
RAID 10RAID 1 + RAID 0 (4 disques)Rapide ET redondantCoût élevé
🏦 Exemple : Serveur d'une banque

Une banque utilise RAID 10 pour ses serveurs : elle a besoin de haute performance (RAID 0) ET de sécurité des données (RAID 1). Si un disque tombe en panne, les données sont intactes sur le miroir.

6.3 — Partitionnement et formatage
Partitionnement : division d'un disque dur en plusieurs zones indépendantes (partitions).

Systèmes de partitionnement :
• MBR (Master Boot Record) : ancien, max 4 partitions primaires, max 2 To
• GPT (GUID Partition Table) : moderne, 128 partitions, disques > 2 To

Systèmes de fichiers (formatage) :
• NTFS : Windows (permissions, journalisation)
• ext4 : Linux (rapide, journalisé)
• FAT32 : compatible tous OS, max 4 Go par fichier
• exFAT : clés USB, cartes SD, grands fichiers

D RAID 10
RAID 10 combine le striping (RAID 0 → performance) et le mirroring (RAID 1 → redondance). C'est le plus robuste mais aussi le plus coûteux.
2. Quel système de fichiers est utilisé par défaut sous Windows ?
A ext4
B FAT32
C NTFS
D exFAT
NTFS (New Technology File System) est le système de fichiers standard de Windows. Il supporte les permissions, la journalisation et les grands fichiers.
✏️ Exercices — Leçon 6
Exercice 6.1

Niveau RAID choisi : RAID 10

13.1 — Définition et principe du CPL
💡 CPL = Courant Porteur en Ligne
Le CPL (Courant Porteur en Ligne), aussi appelé PLC (Power Line Communication) en anglais, est une technologie réseau qui utilise le réseau électrique existant pour transmettre des données numériques.
Le principe est simple : superposer un signal numérique haute fréquence au signal électrique basse fréquence (50 Hz) qui alimente normalement les appareils électriques. Un adaptateur CPL branché dans une prise électrique détecte ce signal et en extrait les données réseau.
13.2 — Comment fonctionne le CPL ?
🔌 Fonctionnement étape par étape

1. Émission : L'adaptateur CPL n°1 (branché au routeur via câble RJ45) reçoit les données réseau et les convertit en signal haute fréquence (1–30 MHz)
2. Transmission : Ce signal voyage sur le câble électrique de la maison/bâtiment
3. Réception : L'adaptateur CPL n°2 (dans une autre pièce) capte le signal haute fréquence
4. Conversion : Il convertit ce signal en données réseau et les envoie via câble RJ45 au PC

• Stable et moins sensible aux interférences que le Wi-Fi
• Bonne portée (jusqu'à 300 m en théorie)

• Ne traverse pas les compteurs électriques différents
• Les multiprises parasites (filtres) bloquent le signal
• Débit réel souvent inférieur au débit théorique
• Perturbations possibles par les appareils électriques

B WPA
C WPA2

✏️ Exercices — Leçon 14
Exercice 14.1
Un responsable informatique doit choisir entre déployer du Wi-Fi 5 (802.11ac) ou du Wi-Fi 6 (802.11ax) dans une école de 500 élèves utilisant simultanément des tablettes. Comparez les deux normes et conseillez-le.
Comparaison :

Wi-Fi 5 (802.11ac) : Bande 5 GHz uniquement, débit max 3.5 Gbps, bonne performance mais peut saturer avec beaucoup d'appareils simultanés.

Wi-Fi 6 (802.11ax) : Bandes 2.4/5/6 GHz, débit max 9.6 Gbps, technologie OFDMA qui gère mieux les connexions simultanées multiples, meilleure efficacité énergétique pour les appareils.

Conseil : Wi-Fi 6 est fortement recommandé dans ce contexte. Avec 500 élèves connectés simultanément, la technologie OFDMA du Wi-Fi 6 permet de servir plusieurs appareils en même temps sans dégradation notable des performances. C'est spécifiquement pour ce scénario (forte densité d'appareils) que Wi-Fi 6 a été conçu.
Exercice 14.2
Listez 4 bonnes pratiques de sécurité pour protéger un réseau Wi-Fi professionnel.
1. Utiliser WPA3 ou WPA2-Enterprise avec authentification via serveur RADIUS pour les réseaux professionnels.
2. Changer le SSID par défaut : ne pas laisser "Livebox-xxxx" ou "TP-Link_xxxx" qui indique la marque du routeur aux attaquants.
3. Créer un réseau invité séparé (VLAN) pour les visiteurs, isolé du réseau interne.
4. Mettre à jour régulièrement le firmware du routeur/point d'accès pour corriger les failles de sécurité.
5. Désactiver le WPS (Wi-Fi Protected Setup) car il présente des vulnérabilités connues.

1.1 — Les équipements de réception
Les équipements de réception permettent de capter les signaux audiovisuels (son et image) diffusés par voie hertzienne, câble ou satellite. On distingue :
• L'antenne : capte le signal électromagnétique (VHF/UHF pour la TNT, parabole pour le satellite)
• Le tuner / démodulateur : sélectionne et démodule le canal voulu
• Le décodeur (set-top box) : déchiffre le signal numérique et le restitue
• Le téléviseur : affiche l'image et restitue le son
💡 Définition
Un équipement de réception est l'ensemble des dispositifs permettant de capter, sélectionner, démoduler et restituer un signal audiovisuel transmis à distance.

1.3 — Le tuner : rôle et fonctionnement
Le tuner (ou syntoniseur) sélectionne la fréquence porteuse du canal désiré parmi toutes celles captées par l'antenne. Il comprend :
• Un filtre d'entrée : élimine les canaux non désirés
• Un oscillateur local : génère la fréquence de référence
• Un mélangeur : produit la fréquence intermédiaire (FI = canal − fosc)
• Un amplificateur FI : amplifie le signal à fréquence intermédiaire
📺 Exemple concret

Un téléviseur reçoit les canaux UHF entre 470 MHz et 862 MHz. Le tuner sélectionne par exemple le canal 21 (474 MHz), le mélange avec l'oscillateur local pour produire une FI de 38,9 MHz, puis amplifie et démodule ce signal.

2.1 — Principe de fonctionnement d'une caméra
Une caméra convertit une scène lumineuse en signal électrique vidéo. Elle comprend :
• L'objectif : concentre la lumière sur le capteur
• Le capteur (CCD ou CMOS) : convertit la lumière en signal électrique
• Le circuit de traitement : amplifie, filtre et numérise le signal
• Le codeur vidéo : compresse et encode le signal (MPEG, H.264...)

CCD (Charge-Coupled Device) : meilleure qualité d'image, moins de bruit, plus consommateur d'énergie. CMOS : moins cher, plus rapide, faible consommation — utilisé dans la quasi-totalité des smartphones et caméras modernes.

Espace = Débit (MB/s) × Durée (s)
3.3 — Enregistrement numérique de la vidéo
Le signal vidéo numérique est échantillonné, quantifié puis compressé avant stockage. Le format d'échantillonnage chromatique le plus courant est le 4:2:0 (TV) ou 4:2:2 (production pro).
📺 Exemple concret

Une vidéo 4K H.264 à 50 Mbits/s nécessite environ 375 MB par minute. Pour 1 heure de tournage, il faut 22,5 Go. Un SSD de 256 Go permet environ 11 heures d'enregistrement en 4K H.264.

4.1 — Principes de la transmission audiovisuelle
La transmission audiovisuelle consiste à acheminer un signal son/image d'un point à un autre par un canal de transmission. Les étapes sont :
• Codage/compression : réduction du débit (MPEG, H.264, H.265)
• Modulation : adaptation du signal au canal de transmission
• Transmission : envoi via le support (hertzien, câble, satellite, fibre)
• Démodulation et décodage : reconstitution du signal à la réception
💡 Modulation
La modulation consiste à superposer le signal utile (modulant) sur une porteuse HF pour le transmettre à distance. Types : AM (amplitude), FM (fréquence), PM (phase), QAM (quadrature).

• Tête de réseau câble : regroupe les signaux et les distribue sur le réseau

5.1 — Le cryptage (chiffrement) du signal
Le cryptage protège le signal audiovisuel contre la réception non autorisée. Il consiste à rendre le signal inintelligible sans clé de déchiffrement.
SystèmeMéthodeUtilisation

5.2 — L'incrustation (chroma key)
L'incrustation consiste à remplacer une couleur de fond (vert ou bleu) par une autre image en temps réel. C'est la technique du fond vert (green screen).
Principe : le fond uni (vert ou bleu) est détecté par le mélangeur vidéo. Tous les pixels de cette couleur sont remplacés par les pixels de l'image de remplacement (météo, décor virtuel).
📺 Exemple concret

Le présentateur météo se tient devant un fond vert. Le mélangeur vidéo remplace tous les pixels verts par la carte météorologique numérique. Le résultat affiché à la télévision montre le présentateur devant la carte, en temps réel.

5.3 — L'adressage
L'adressage est une technique qui permet d'envoyer des informations spécifiques à un décodeur particulier. Il est utilisé pour :
• Activer/désactiver des abonnements à distance
• Envoyer des messages personnalisés à un décodeur
• Gérer les droits de visionnage (PPV, Pay-Per-View)
• Mettre à jour les clés de cryptage à distance (EMM : Entitlement Management Messages)

1. Quelle couleur de fond est principalement utilisée pour l'incrustation en TV ?
A Rouge
B Blanc
C Vert (ou bleu)
D Noir
L'incrustation utilise principalement le fond vert (green screen) car cette couleur est la plus éloignée des tons de peau humaine, facilitant la détection par le mélangeur. Le bleu est une alternative classique.
2. Que signifie l'acronyme EMM dans le système d'accès conditionnel ?
A Electronic Music Module

C Encrypted Multimedia Message
D Error Management Module
EMM = Entitlement Management Message. Ce sont des messages adressés individuellement à chaque décodeur pour mettre à jour les droits d'accès et les clés de déchiffrement.
3. Qu'est-ce que le PPV (Pay-Per-View) ?
A Un type de modulation satellite
B Un système de paiement à l'unité pour regarder un événement
C Un format de compression vidéo
D Un type de câble HDMI
PPV (Pay-Per-View) = paiement à la vue. L'abonné paie pour débloquer un événement précis (match de foot, film en avant-première) via le système d'adressage de son décodeur.
✏️ Exercices — Chapitre 5
Exercice 5.1
Expliquez comment fonctionne le système d'accès conditionnel (CAS) pour protéger une chaîne TV payante, depuis l'encodage chez l'opérateur jusqu'à la restitution chez l'abonné.
Chez l'opérateur (tête de réseau) :
1. Le signal vidéo est chiffré avec une clé de contrôle (CW - Control Word) qui change toutes les 10s.
2. La CW est elle-même chiffrée dans un ECM (Entitlement Control Message) envoyé avec le signal.
3. Des EMM sont envoyés périodiquement pour mettre à jour les droits de chaque abonné.

Chez l'abonné (décodeur + carte à puce) :
4. La carte à puce du décodeur reçoit l'ECM, vérifie les droits de l'abonné et déchiffre la CW.
5. La CW est transmise au décodeur qui déchiffre le signal vidéo.
6. Le signal est décodé (MPEG) et restitué sur l'écran.

Les télétextes

6.1 — Définition et historique
Le télétexte est un système de diffusion de textes et de graphiques simples via le signal de télévision. Il a été développé dans les années 1970 (Ceefax au Royaume-Uni, Antiope en France).
💡 Définition
Le télétexte utilise les lignes de suppression verticale (VBI - Vertical Blanking Interval) du signal TV analogique pour transmettre des données textuelles sans perturber l'image principale.
6.2 — Fonctionnement technique
En analogique, les données télétexte sont insérées dans le VBI (lignes 6 à 22 du signal PAL). Chaque page télétexte peut contenir 24 lignes de 40 caractères.

En numérique (DVB), le télétexte est encapsulé dans le flux MPEG-2 via des paquets PES (Packetized Elementary Stream) dédiés. Il coexiste avec les sous-titres DVB.

Ch.12

La CAG (Commande Automatique de Gain)

CAGAGCRégulation
▼
12.1 — Définition et principe
La CAG (Commande Automatique de Gain), ou AGC (Automatic Gain Control) en anglais, est un système qui ajuste automatiquement le gain d'un amplificateur pour maintenir un niveau de sortie constant, quelle que soit la variation du signal d'entrée.
💡 Principe
La CAG compare la tension de sortie à une référence. Si la sortie est trop forte → le gain diminue. Si la sortie est trop faible → le gain augmente. C'est une boucle de contre-réaction negative.
12.2 — Applications de la CAG
• Tuner TV et radio : compense les variations de niveau des canaux reçus
• Téléphone portable : maintient un niveau sonore constant à l'oreille
• Caméra vidéo : CAG audio — maintient le niveau sonore d'enregistrement
• Radar : compense les échos de distances variables
• Modem : adapte le gain selon la qualité de la ligne
12.3 — CAG dans un tuner TV
Dans un récepteur TV, la CAG maintient le signal FI (Fréquence Intermédiaire) à un niveau constant :
1. Le détecteur mesure le niveau du signal FI
2. La tension CAG est générée (proportionnelle à l'écart)
3. Cette tension commande les amplificateurs HF et FI via des varicaps ou FET
4. Le gain s'ajuste automatiquement
📺 Exemple concret

Une antenne TV reçoit des signaux variant de 40 dBµV (canal éloigné) à 80 dBµV (canal local). Sans CAG, les canaux forts satureraient l'étage d'entrée. La CAG réduit automatiquement le gain pour les signaux forts et l'augmente pour les signaux faibles, maintenant un niveau constant de ~60 dBµV en sortie.

16.1 — Architecture d'un système satellite
Un système de télédistribution par satellite comprend :
• Le segment spatial : satellite géostationnaire à 36 000 km
• Le segment sol montant (uplink) : station terrestre émettrice
• Le segment sol descendant (downlink) : récepteurs chez les abonnés
💡 Satellite géostationnaire
Un satellite géostationnaire orbite à 36 000 km d'altitude sur l'équateur. Sa vitesse de rotation est égale à celle de la Terre → il semble immobile dans le ciel. Une antenne fixe peut donc le viser en permanence.
16.2 — Le LNB (Low Noise Block)
Le LNB est placé au foyer de la parabole. Il effectue :
• Amplification à faible bruit : amplifie le signal satellite reçu (très faible : ~10⁻¹² W)
• Conversion de fréquence : Ku-Band (10,7-12,75 GHz) → FI (950-2150 MHz)
• Cette FI peut ensuite transiter sur câble coaxial jusqu'au décodeur
📐 Fréquence intermédiaire LNB
FI = f_satellite − f_OL  (OL universel = 9750 MHz ou 10600 MHz)
16.3 — Standard DVB-S2 et réception collective
Le standard DVB-S2 (Digital Video Broadcasting – Satellite, 2ème génération) est le standard actuel pour la TV par satellite. Il utilise la modulation QPSK/8PSK/16APSK avec le code correcteur d'erreur LDPC + BCH.
La réception collective (SMATV) distribue un ou plusieurs bouquets satellite à plusieurs logements via :
• LNB multi-sorties ou Quattro LNB + multicommutateur
• Amplificateur tête de réseau + dérivateurs + répartiteurs
• Câble coaxial 75Ω jusqu'à chaque logement
📡 Exemple concret

Un immeuble de 40 appartements dispose d'une seule parabole orientée vers Astra 19,2°E. Un Quattro LNB fournit 4 sorties (Bas/Haut × Horizontal/Vertical). Un multicommutateur 4×40 distribue les 40 décodeurs, chacun pouvant accéder à n'importe quel transpondeur de manière indépendante.

5.1 — Structure et fonctionnement
Le TUJ (Transistor Unijonction), ou UJT (Unijunction Transistor), est un composant à trois électrodes : Émetteur (E), Base 1 (B1) et Base 2 (B2). Il possède une seule jonction PN, contrairement au BJT.

En régime non linéaire, l'AOP fonctionne sans contre-réaction négative (ou avec contre-réaction positive). La sortie est saturée à :

• Buzzer : émet un son quand il reçoit l'alimentation.

3. MOSFET N-channel (Commutation/Puissance) : Commandé par la tension VGS issue de l'AOP. Il module le courant dans le moteur (souvent en commutation PWM).

4. Moteur DC (Sortie/Actionneur) : Reçoit le courant modulé et tourne à une vitesse proportionnelle à la tension moyenne appliquée.

Conclusion : Ce circuit permet de faire varier la vitesse du moteur en jouant sur le potentiomètre. Plus la consigne est élevée, plus le MOSFET conduit longtemps, plus le moteur tourne vite.
